\subsubsection*{template$<$typename Tlabel, bool directed$>$\newline
class Neo\+FOAM\+::device\+Adjacency$<$ Tlabel, directed $>$}

Represents an adjacency graph, for both directed and undirected variants. 


\begin{DoxyTemplParams}{Template Parameters}
{\em Tlabel} & The type of the labels in the adjacency matrix. \\
\hline
{\em directed} & A boolean value indicating whether the adjacency matrix is directed or not.\\
\hline
\end{DoxyTemplParams}
The {\ttfamily \mbox{\hyperlink{classNeoFOAM_1_1deviceAdjacency}{device\+Adjacency}}} class represents an adjacency graph, which records the connectivity relationship between nodes through edges. The graph stores specifically the connectivity between nodes in the graph. The template parameter {\ttfamily Tlabel} specifies the type of the labels int, uint32, etc. The {\ttfamily directed} parameter indicates whether the graph is directed or undirected. In the undirected variant the graph edges are considered to connect both nodes (of the edge), and therefor no distiction is made interms of direction between the 0 and 1 node of the edge. Thus the graph is symmetric, for example, if node 10 is connected to node 20 then node 20 is connected to 10. In the directed variant, an edge connects the 0 node to the 1 node, and therefor the graph is not symmetric. For example, if node 10 is connected to node 20, the reverse is not true.

A word no memory layout\+: To optimise memory access the data in the class is stored \textquotesingle{}flat\textquotesingle{} with an adjacency View and an offset View. The former contains the connections of the graph while the later stores the start and end position (offset to) of each nodes\textquotesingle{} connectivity in the adjacency container. Within each node\textquotesingle{}s connectivity the connections are sorted in accending order. Finally, the offset container is one greater than the size of the adjacency container, to faciliate slightly more lazy programing -\/ since the first element will always be zero.

\begin{DoxyNote}{Note}
An directed graph can therefor be used to represent the connection between two different \textquotesingle{}types/classes\textquotesingle{} of nodes rather than the same. For example, a graph could be used to represent the connection between cells and faces.

The undirected graphs allow for nodes to be self referenced/connected.
\end{DoxyNote}
